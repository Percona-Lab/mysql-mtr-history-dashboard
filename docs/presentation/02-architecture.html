Architecture

A three-stage pipeline
from builds to dashboards

Jenkins build data flows through a canonical JSON layer, gets exported as OpenMetrics, and lands in Prometheus for Grafana to visualize.

Stage A

Collect

Fetch build metadata and download 19 JUnit XML artifacts per build. Parse, merge duplicates, resolve test identity. Write one canonical JSON per build.

Jenkins REST API JUnit XML Pydantic v2
Stage B

Transform

Convert per-build JSON to OpenMetrics text format. Generate 5 metric types with full label sets. Merge per-build files into a single sorted bundle.

OpenMetrics 5 metrics 170K samples
Stage C

Ingest

Rsync the merged OpenMetrics file to the Hetzner host. Use promtool to create TSDB blocks directly, then reload Prometheus. Safe for re-runs.

promtool Prometheus TSDB SSH + rsync
Visualize

Dashboard

Grafana failure matrix shows tests vs. builds. Clickable cells deep-link to Jenkins build pages, test reports, and worker logs. Filters by platform, branch, fork.
